Why LiveBoard

Built for developers
who own their workflow

No accounts. No subscriptions. No vendor telling you how to organize your work.

#

Markdown is the database

Each board is a single .md file. Columns are H2 headings. Cards are list items. Human-readable without any tooling, editable in any text editor. No proprietary format, no binary blobs, no SQLite files to corrupt — just the Markdown you already know.

⊘

No database. No provider.

Zero external dependencies. No cloud account, no API key, no sync server. LiveBoard reads and writes flat files in a folder. That's it.

⊕

Your data. You own it.

Files live on your disk. You decide where they go, who sees them, and how long they exist. No data mining, no telemetry, no "we updated our privacy policy" emails.

⇄

Sync your way

Git, iCloud, Dropbox, Syncthing, rsync, a USB stick — if it can move files, it can sync your boards. Git integration is built in, everything else just works by nature.

◈

Single binary. Cross platform.

Written in Go. One binary, no runtime dependencies. Runs on macOS, Linux, Windows. Download it, run it, done. Compiles to a ~15MB executable.

◧

Web UI ready. TUI coming soon.

The built-in web UI runs at localhost:7070 with drag-and-drop Kanban, search, and real-time updates via SSE. A terminal UI with Bubbletea is next on the roadmap.
