# LiveBoard REST API v1

## Overview

All v1 endpoints are prefixed with `/api/v1`. A version-probe endpoint lives at `/api/versions` (no version prefix).

| Property | Value |
|---|---|
| Base URL | `http://localhost:7070` |
| Content-Type | `application/json` (requests and responses) |
| Authentication | None (v1 is local-only) |

All mutating endpoints accept and return `application/json`. The SSE endpoint (`/api/v1/events`) returns `text/event-stream`.

---

## Error Model

Every error response uses HTTP status 4xx/5xx with the following JSON body:

```json
{
  "error": "human-readable message",
  "code":  "CANONICAL_CODE",
  "status": 404
}
```

### Canonical Codes

| Code | HTTP Status | Meaning |
|---|---|---|
| `NOT_FOUND` | 404 | Board slug or file does not exist |
| `VERSION_CONFLICT` | 409 | `client_version` mismatch (optimistic locking) |
| `ALREADY_EXISTS` | 409 | Board with that name already exists |
| `OUT_OF_RANGE` | 400 | `col_idx` or `card_idx` is out of bounds |
| `INVALID` | 400 | Malformed JSON, missing required field, or invalid value |
| `INTERNAL` | 500 | Unexpected server error |

---

## Optimistic Locking

Every board has a monotonically increasing `version` integer. To prevent lost-update races, mutations require the caller to supply `client_version` — the version the client last saw.

- If `client_version` matches the server version, the mutation proceeds and the board version is incremented.
- If they differ, the server returns `409 VERSION_CONFLICT`.
- Pass `client_version: -1` to bypass the check entirely (use with caution; last-write wins).

The response body of a successful mutation is the full updated `Board` object, which contains the new `version`.

## Endpoints

### GET /api/versions

Version probe. Returns the supported API versions.

**Response 200**

```json
{
  "supported": ["v1"],
  "current": "v1"
}
```

**Example**

```bash
curl http://localhost:7070/api/versions
```

### GET /api/v1/workspace

Returns workspace-level metadata.

**Response 200**

```json
{
  "dir": "/path/to/workspace",
  "board_count": 3
}
```

**Example**

```bash
curl http://localhost:7070/api/v1/workspace
```

---

### GET /api/v1/boards

List all boards in the workspace.

**Response 200** — array of `Board` objects (same shape as `GET /api/v1/boards/{slug}`). Returns `[]` for an empty workspace.

**Example**

```bash
curl http://localhost:7070/api/v1/boards/
```

**Response 200** — `Board` object:

```json

  "version": 5,
  "name": "Demo Board",
  "description": "My demo",
  "icon": "🚀",
  "tags": ["product"],
  "tag_colors": {"product": "#6366f1"},
  "members": ["alice"],
  "list_collapse": [false, false, true],
  "settings": { ... },
  "columns": [
    {
      "name": "Backlog",
      "collapsed": false,

        {
          "title": "Fix login bug",
          "completed": false,
          "tags": ["backend"],
          "assignee": "alice",
```

**Example**

```bash

```

### POST /api/v1/boards/{slug}/mutations

Apply a single mutation to a board. The request body is a flat JSON object with a `type` discriminator field, the `client_version`, and all fields for the chosen op type.

**Request shape**

```json
{
  "client_version": 5,
  "type": "<op_type>",
  ... op-specific fields ...
}
```

**Response 200** — the full updated `Board` object (new `version` already incremented).

**Errors:** `NOT_FOUND`, `VERSION_CONFLICT`, `OUT_OF_RANGE`, `INVALID`, `INTERNAL`

---

#### Op: `add_card`

Fields: `column` (string), `title` (string), `prepend` (bool, optional — default false appends).

```bash
curl -X POST http://localhost:7070/api/v1/boards/demo/mutations \
  -H "Content-Type: application/json" \
  -d '{"client_version":5,"type":"add_card","column":"Backlog","title":"New card","prepend":false}'
```

---

#### Op: `move_card`

Fields: `col_idx` (int), `card_idx` (int), `target_column` (string).

Move card at `[col_idx][card_idx]` to the end of `target_column`.

```bash
curl -X POST http://localhost:7070/api/v1/boards/demo/mutations \
  -H "Content-Type: application/json" \
  -d '{"client_version":5,"type":"move_card","col_idx":0,"card_idx":2,"target_column":"In Progress"}'
```

---

#### Op: `reorder_card`

Fields: `col_idx` (int), `card_idx` (int), `before_idx` (int), `target_column` (string).

Move card to a specific position (`before_idx`) within `target_column`. Set `before_idx` to the length of the target column to append.

```bash
curl -X POST http://localhost:7070/api/v1/boards/demo/mutations \
  -H "Content-Type: application/json" \
  -d '{"client_version":5,"type":"reorder_card","col_idx":0,"card_idx":1,"before_idx":0,"target_column":"Backlog"}'
```

### GET /api/v1/boards/{slug}/settings

Returns the **resolved** (effective) settings for the board — global defaults merged with per-board overrides. All fields are non-null concrete values.

**Response 200** — `ResolvedSettings`:

```json
{
  "show_checkbox": true,
  "newline_trigger": "enter",
  "card_position": "bottom",
  "expand_columns": true,
  "view_mode": "board",
  "card_display_mode": "default",
  "week_start": "sunday"
}
```

**Errors:** `NOT_FOUND`

**Example**

```bash
curl http://localhost:7070/api/v1/boards/demo/settings
```

---

### PUT /api/v1/boards/{slug}/settings

Replace the board's per-board setting overrides. This is a full replacement — omitted fields become `null`, clearing any existing override for that field so it inherits the global default at resolve time.

**Request body** — `BoardSettings` (all fields optional):

```json
{
  "show_checkbox": true,
  "card_position": "top",
  "expand_columns": false,
  "view_mode": "board",
  "card_display_mode": "compact",
  "week_start": "monday"
}
```

**Response 204** — no body.

**Errors:** `NOT_FOUND`, `INVALID`

**Example**

```bash
curl -X PUT http://localhost:7070/api/v1/boards/demo/settings \
  -H "Content-Type: application/json" \
  -d '{"view_mode":"calendar","week_start":"monday"}'
```

### GET /api/v1/events?board={slug}

Server-Sent Events stream for a board. The client receives real-time notifications when the board is mutated.

**Query params:** `board` (required) — board slug.

**Response headers:**

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
```

**Event types:**

| Event | Data | When |
|---|---|---|
| `connected` | `{}` | Immediately on connection |
| `board.updated` | `{"board_id":"demo","version":6}` | After any successful mutation |

The server also emits SSE comment pings (`: ping`) every 15 seconds to keep the connection alive through proxies.

**Example** (curl, streams indefinitely):

```bash
curl -N -H "Accept: text/event-stream" \
  "http://localhost:7070/api/v1/events?board=demo"
```

**Sample stream:**

```
event: connected
data: {}

event: board.updated
data: {"board_id":"demo","version":6}

: ping
```

**Errors:** `INVALID` (missing `board` param) — returned as JSON with status 400 before the stream is opened.
