Why LiveBoard

Built for developers
who own their workflow

No accounts. No subscriptions. No vendor telling you how to organize your work.

#

Markdown is the database

Each board is a single .md file. Columns are H2 headings. Cards are list items. Human-readable, editable in any text editor.

⊘

No database. No provider.

Zero external dependencies. No cloud account, no API key, no sync server. Reads and writes flat files in a folder.

⊕

Your data. You own it.

Files live on your disk. You decide where they go, who sees them, and how long they exist. No telemetry, no data mining.

⇄

Sync your way

Git, iCloud, Dropbox, Syncthing, rsync — if it can move files, it can sync your boards. Git integration built in.

◈

Single binary. Cross platform.

Written in Go. One binary, no runtime deps. macOS, Linux, Windows. Download, run, done.

◧

Web UI with real-time updates

Drag-and-drop Kanban at localhost:7070 with search, themes, calendar view, and SSE collaboration.

👁

Read-only mode

Share boards safely. Read-only mode disables editing, deleting, and drag-and-drop — perfect for dashboards and public displays.

↓

Export to HTML + ZIP

Export your entire workspace as a self-contained ZIP of static HTML files. Share with anyone — no server needed to view.
